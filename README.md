# INSTVNCE_BLOG

A raw developer's log. Not a polished publication ‚ÄĒ an active terminal window into how I think, build, and decide.

---

## What it is

INSTVNCE is a **personal technical blog** built around one idea: every post is an instance ‚ÄĒ a deployed unit of thought. You write it, you fire it, you can terminate it. The interface reflects that.

The aesthetic is neopunk terminal: CRT scanline overlays, holographic post cards with chamfered corners, neon green on cosmic black, monospace everywhere. It's not decoration ‚ÄĒ it's a philosophy. The medium should feel like the work.

![Nav expanded](.github/assets/instance-nav.png)

---

## What makes it unique

Most dev blogs look like Medium. This one looks like a monitoring dashboard.

- **The architect voice** ‚ÄĒ posts are `ACTIVE_LOG` instances. The input says `AIM`. Delete says `TERMINATE`. The tone is not content marketing; it's raw operational log.
- **CRT + holographic aesthetic** ‚ÄĒ a custom design system built from scratch: animated scanlines, clip-path chamfered cards, multi-layer holographic backgrounds, golden-ratio bottom accent lines that expand on hover.
- **Color-coded operational states** ‚ÄĒ inputs shift hue dynamically as you type. Click `[EDIT]` and the form enters heavy-load yellow: text, border, and accent line all switch to signal you're modifying something live. `AIM` morphs to `FIRE` in red on hover. Green = deploy. Yellow = modify. Red = destroy. No labels needed.
- **3D braces** ‚ÄĒ the `{ }` framing the input form are real Three.js geometry: metallic material, triple-point neon lighting, floating animation. WebGL just to frame a text input.
- **Honest format** ‚ÄĒ no SEO filler, no "in this post I will". Just the thought, the decision, the thing learned.
- **Full control** ‚ÄĒ self-hosted, no CMS, no platform risk. You own every instance.

## Stretch Goals

### `[?]` Raw Markdown Toggle ‚ÄĒ per post card

A small round button positioned outside and adjacent to each post card, styled with the holographic black design system (chamfered clip-path, gradient border, cosmic bg). It contains a `?` glyph.

**Behavior:**
- Clicking it toggles the post body between:
  - **Rendered view** ‚ÄĒ ReactMarkdown output (current default)
  - **Raw view** ‚ÄĒ the original markdown source, displayed in a monospace `<pre>` block with the same terminal styling as the input textarea

**Design notes:**
- Button is round, ~32px, positioned top-right outside the card boundary (absolute, negative offset)
- Active state: button glows brighter / border shifts to cyan to signal raw mode is on
- Toggle is per-card, not global ‚ÄĒ each instance manages its own view state
- No backend changes needed ‚ÄĒ `post.content` already stores the raw markdown string
